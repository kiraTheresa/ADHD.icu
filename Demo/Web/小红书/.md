```text
你是一个资深前端工程师，请帮我实现一个完整可运行的 Web Demo 项目（无需后端）。

# 项目目标

实现一个类似“小红书信息流布局”的网页，但内容不是图片或文字，而是“低认知负担的动态色块卡片”。

这个项目的核心是：
- 模拟信息流的“滑动扫描体验”
- 但去除所有语义信息（无文字、无图片内容）
- 只保留视觉刺激（颜色 + 轻微动态）

# 技术要求

- 使用：HTML + CSS + JavaScript（原生即可，不强制框架）
- 可以使用 Canvas 或 CSS 动画实现动态效果
- 项目必须可以直接用浏览器打开 index.html 运行
- 代码结构清晰，适当模块化
- 不要使用复杂依赖（不要webpack、不要构建工具）

# 页面布局（参考小红书）

实现一个双列瀑布流布局（类似小红书）：

- 页面背景：浅灰色
- 中间内容区域居中
- 左右两列卡片流
- 卡片有间距（类似8px~12px）
- 卡片为圆角矩形

# 卡片内容设计（核心）

每个卡片必须满足：

1. 完全无文字、无图片
2. 内部为“动态色块区域”，例如：
   - 渐变色（linear-gradient）
   - 缓慢变化的颜色
   - 轻微移动的几何形状（方块 / 圆形）
3. 动画必须：
   - 缓慢（不能刺激）
   - 平滑（ease-in-out）
   - 不要闪烁

建议实现：
- 每个卡片随机生成：
  - 主色调（低饱和度）
  - 动画类型（渐变 / 漂移 / 呼吸效果）
  - 动画速度（可控）

# 用户交互（简化版）

实现以下基础交互：

1. 无限滚动（模拟信息流）
   - 滚动到底部自动加载新卡片

2. 全局“刺激强度控制”滑块（页面顶部）
   - 控制动画速度
   - 控制颜色对比度
   - 控制动态幅度

3. 可选：
   - 一个“极简模式”按钮（降低动画）

# 视觉风格要求

整体风格：
- 极简
- 无干扰
- 类似“放空工具”

颜色：
- 避免高饱和（不要刺眼）
- 偏柔和（莫兰迪色系）

动画：
- 不要出现快速闪烁
- 不要强对比跳变

# 代码结构建议

- index.html
- style.css
- script.js

JS中建议模块：
- createCard()：生成卡片
- applyAnimation()：给卡片添加动画
- loadMore()：加载更多
- updateStimulusLevel()：控制刺激强度

# 输出要求

请直接输出完整代码：
- index.html
- style.css
- script.js

确保复制后可以直接运行

# 额外优化（如果可以）

- 使用 requestAnimationFrame 优化动画
- 动画参数与“刺激强度”绑定
- 保持性能流畅（至少100个卡片不卡顿）
```
